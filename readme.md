# GitHub Flow Workshop Cho Sinh Viên Làm Dự Án Tốt Nghiệp

Series 3 buổi chia sẻ thực chiến giúp sinh viên hiểu cách teamwork bằng Git/GitHub khi làm project nhóm, đồ án tốt nghiệp hoặc sản phẩm thực tế.

Workshop không tập trung học thuộc lệnh Git, mà tập trung vào trải nghiệm cảm giác làm việc nhóm thật:

- nhận task
- chia việc
- code song song
- review code
- merge conflict
- teamwork khi deadline tới gần

Mục tiêu cuối cùng:

> Giúp sinh viên biết cách làm project team mà không “toang” source code.

---

# Đối Tượng Phù Hợp

- Sinh viên làm dự án tốt nghiệp
- Sinh viên làm project nhóm
- Người mới học Git/GitHub
- Team chưa có workflow rõ ràng
- Người từng:
  - mất code
  - sửa đè code nhau
  - gửi file `.zip`
  - push thẳng vào `main`
  - sợ pull code

---

# Hình Thức Workshop

- Học theo kiểu thực chiến
- Làm mini project theo team
- Mentor đóng vai reviewer/team lead
- Sinh viên tự tạo lỗi và tự xử lý
- Không quá lý thuyết
- Mô phỏng môi trường làm việc thật

---

# Mini Project Sử Dụng

Workshop sẽ dùng một project nhỏ để sinh viên cùng phát triển theo nhóm.

Ví dụ:

- Website đặt vé xem phim
- Website quản lý phòng trọ
- Website đặt vé xe
- Website đặt lịch sân bóng

Project sẽ được chia thành nhiều feature nhỏ để các thành viên làm song song và dễ phát sinh conflict thực tế.

# Nội Dung 3 Buổi

---

# Buổi 1 — Code Một Mình Khác Gì Code Theo Team?

Buổi đầu tiên giúp sinh viên hiểu sự khác nhau giữa làm bài cá nhân và làm project nhóm.

Sinh viên sẽ bắt đầu trải nghiệm quy trình teamwork cơ bản:

- nhận task
- chia việc
- tạo branch riêng
- commit code
- push code lên GitHub
- tạo Pull Request
- review cơ bản

Ngoài việc học GitHub Flow, sinh viên còn bắt đầu cảm nhận:

- vì sao không nên code trực tiếp trên `main`
- vì sao cần review trước khi merge
- vì sao teamwork cần quy tắc chung

Kết thúc buổi:

- mỗi nhóm merge được feature đầu tiên vào project chung
- lần đầu thấy code của nhiều người chạy cùng nhau

---

# Buổi 2 — Khi Conflict Bắt Đầu Xuất Hiện

Sau khi teamwork bắt đầu thật sự, các vấn đề phổ biến sẽ xuất hiện.

Sinh viên sẽ trực tiếp trải nghiệm:

- sửa đè code nhau
- merge conflict
- reviewer reject Pull Request
- requirement thay đổi giữa chừng
- teamwork bắt đầu “căng”

Mục tiêu buổi này không phải tránh lỗi.

Mà là:

> hiểu vì sao conflict xuất hiện và cách xử lý khi teamwork thật.

Sinh viên sẽ học cách:

- pull code đúng thời điểm
- resolve conflict
- sửa PR theo review
- giao tiếp khi làm việc nhóm

Kết thúc buổi:

- resolve conflict thành công
- hiểu cảm giác teamwork thực tế khi project bắt đầu lớn hơn

---

# Buổi 3 — Làm Sao Để Team Code Không Toang?

Sau khi đã trải nghiệm đủ “drama”, buổi cuối sẽ tập trung tối ưu workflow teamwork.

Sinh viên sẽ thực hành:

- viết commit message rõ ràng
- chia task hợp lý hơn
- review code hiệu quả
- giảm conflict
- xây dựng workflow ổn định hơn

Ngoài ra sinh viên sẽ:

- nhìn lại các lỗi ở 2 buổi trước
- cải thiện cách phối hợp giữa các thành viên
- chạy lại mini project theo quy trình tốt hơn

Mục tiêu cuối cùng:

> teamwork mượt hơn, ít conflict hơn và dễ maintain project hơn.

---

# Điều Sinh Viên Nhận Được Sau Workshop

Sau 3 buổi, sinh viên sẽ:

- hiểu GitHub Flow thực tế
- biết làm việc nhóm bằng Git/GitHub
- biết cách review và merge code
- biết xử lý conflict cơ bản
- hiểu vai trò của workflow trong teamwork
- có trải nghiệm gần với môi trường doanh nghiệp hơn

Quan trọng nhất:

> Sinh viên nhớ conflict và PR reject lâu hơn nhớ định nghĩa Git.
